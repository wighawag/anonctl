# Endpoint share-class is a port heuristic (operator-overridable), and the endpoint is credential-free at rest

An account's endpoint has a cross-user share-class (`tor-shared` vs `socks-peruser`) that decides whether it is safe to share across accounts. We classify it by a HEURISTIC on the address, not by probing the running proxy: a loopback SOCKS on a Tor-conventional port (9050/9150) is `tor-shared`, everything else is conservatively `socks-peruser`. We chose this because a SOCKS proxy does not announce that it is Tor, so there is no honest way to CONFIRM `tor-shared` from a dial (netcage's detect-proxy makes the same never-label-the-provider call); a wrong `tor-shared` label would be a dangerous lie (it would let two accounts share a single-identity endpoint and become cross-identifiable). The heuristic fails SAFE: an unrecognised port defaults to `socks-peruser`, which triggers the one-account-only refusal rather than a false sharing guarantee. The operator can override the class explicitly (the class is a parameter to `endpoint.Parse`, not re-derived inside it), so a real per-user Tor on a non-standard port, or a deliberately-shared endpoint, is still expressible.

Considered and rejected: (a) actively probing the proxy to detect Tor (impossible to do honestly, and issuing a CONNECT would egress); (b) making the operator ALWAYS declare the class (worse ergonomics for the default `anonctl add` case, which must work out of the box on the local Tor SocksPort per story 4). The heuristic gives the common case a correct default while keeping the unsafe direction (assuming isolation that is not there) behind an explicit operator choice.

Separately, the persisted endpoint is CREDENTIAL-FREE at rest by construction: `endpoint.Parse` REFUSES an endpoint URL carrying embedded `user:pass@` credentials (`ErrCredentialedEndpoint`) rather than silently dropping them. This mirrors netcage's `ErrCredentialedProxyNotPersisted` config hygiene and is load-bearing here because the anonctl marker (`/etc/anonctl/<account>.json`) is world-readable and must never hold a secret. The per-account isolation username anonctl dials with (`<account>@`, empty password, the Tor `IsolateSOCKSAuth` knob) is DERIVED at dial time (`Endpoint.IsolationUsername`), never stored; a real authed upstream proxy's credentials stay in that proxy's own config, outside anonctl.
