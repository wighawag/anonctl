# LiveParams.Exempt is not populated by verifyParams (both split-tunnel-tight and lan-exemption-not-a-dns-hole depend on it)

2026-07-07: `main.go` `verifyParams` (around line 267) never sets `verify.LiveParams.Exempt`, so BOTH the pre-existing `split-tunnel-tight` assertion and the new `lan-exemption-not-a-dns-hole` assertion are skipped at runtime (they only run when `p.Exempt != ""`). Exemptions are also not persisted in `internal/accountconfig`. This is a pre-existing gap (the exemption persistence + verify-wiring is a separate concern from generating the exemption rules); noting it so the persistence/wiring task threads the configured exemption through to `verifyParams`. Both integration tests drive the assertion directly via `verify.RunVerify` with `Exempt` set, so the assertion logic and live probe are proven regardless.
